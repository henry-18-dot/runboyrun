# ComfyUI Layered Asset Workflow

ComfyUI is a candidate generator, not the source of layer structure. The layer structure belongs to the repo taxonomy, manifest, review site, and renderer.

## Workflow

1. Pick one biome, one layer role, and one output size.
2. Generate only that layer. Do not ask a prompt to produce a full scene and split it later.
3. Keep every output in `public/assets/candidates/visual`.
4. Register metadata in `src/assets/candidateManifest.json`.
5. Build the review site with `npm run assets:review`.
6. Review isolated layer, 3x seam, composition safety, and metadata warnings.
7. Only approved assets may later move into approved manifests.

## Prompt Rule For Future AI Layers

Prompts should say the target layer and exclude all other layers. Example:

```text
pixel art side-scrolling game layer, grassland biome, playable ground strip only,
clear hoof contact surface, horizontal seamless tile, low visual noise,
no sky, no mountains, no character, no horse, no foreground, no text
```
