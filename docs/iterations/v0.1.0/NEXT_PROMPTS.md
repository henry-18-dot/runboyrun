## 项目核心与设计风险

请评估 v0.2 应优先验证哪一个玩家可感知结果。重点看：核心体验是否清晰、哪些内容应放进 Not now、资产管线会不会让项目变成素材展示。

## 项目结构与技术风险

请评估 v0.2 的技术边界。重点看：visual taxonomy、Asset Orchestrator v1 是否只做 visual、ComfyUI watcher vs API runner、approved manifest 接入风险。

## 音乐风格设计与规划

请评估音乐是否应继续延后自动化。重点看：长期陪伴感、低侵入循环、音乐与 ambience 的关系，以及 v0.2 是否只需要音乐规范。

## 美术规范

请提出 visual asset taxonomy 的最低可用版本。重点看：biome、layer、role、尺寸、循环、像素风一致性、ComfyUI preset 输入输出。

## 音效规划

请评估音效管线是否应延后。重点看：ambience/hoofstep 分类、loop point、LUFS、one-shot 变体、audio manifest 审查字段。
